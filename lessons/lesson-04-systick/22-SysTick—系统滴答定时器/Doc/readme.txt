【*】程序简介

- 工程名称：SysTick 系统滴答定时器
- 实验平台：野火 STM32-F103 指南者开发板
- ST 固件库版本：3.5.0

【!】功能简介：
使用 Cortex-M3 内核自带的 SysTick 定时器产生 1ms 周期中断。
中断服务函数每 1ms 累加一次系统 tick，应用层通过 Delay_ms() 实现毫秒级阻塞延时。

【!】实验操作：
编译并下载程序到开发板并复位。
串口会先打印 lesson-04-systick start。
随后蓝灯每 500ms 翻转一次，串口每 500ms 打印当前 tick。

/*********************************************************************************************/

【*】引脚分配

USART1：
	TX <----> PA9
	RX <----> PA10

LED：
	LED_R <----> PB5
	LED_G <----> PB0
	LED_B <----> PB1

对应 LED 引脚输出低电平则 LED 亮。

/*********************************************************************************************/

【&】程序描述

<bsp_systick.h>
	1. 声明 SysTick_Init()
	2. 声明 SysTick_GetTick()
	3. 声明 Delay_ms()
	4. 声明供中断服务函数调用的 SysTick_IncTick()

<bsp_systick.c>
	1. 使用 SystemCoreClockUpdate() 更新系统主频变量
	2. 使用 SysTick_Config(SystemCoreClock / 1000) 配置 1ms 中断
	3. 在 SysTick_IncTick() 中累加毫秒 tick
	4. Delay_ms() 根据 tick 差值实现阻塞式毫秒延时

<stm32f10x_it.c>
	1. SysTick_Handler() 每 1ms 被硬件自动调用一次
	2. 在 SysTick_Handler() 中调用 SysTick_IncTick()

<main.c>
	1. 初始化 LED
	2. 初始化 USART1
	3. 初始化 SysTick
	4. 在 while(1) 中翻转蓝灯、打印 tick、延时 500ms
